# ABC Method

This repository uses a simple heuristic called ABC to score each lesion image.
The goal is not to replace a trained model, but to provide an explainable baseline
that highlights suspicious cases using three interpretable visual criteria:

- A: asymmetry
- B: border irregularity
- C: color variegation

The implementation lives in `scripts/evaluate_abc.py`, and the visualization panels
are generated by `scripts/visualize_abc.py`.

## 1. Inputs

The pipeline works with two files for each case:

- the dermoscopy image from `images/`
- the binary segmentation mask from `masks/`

The mask is thresholded to a binary image, so pixels inside the lesion are 255
and pixels outside are 0.

## 2. Mask Application

The mask is the reference object for all three criteria.

In the visualization, it is used in two ways:

- as a red overlay on top of the original image, so the lesion extent is visible
- as the region of interest for asymmetry, border, and color analysis

If no mask is found, the visualization falls back to empty panels, but the normal
workflow assumes a valid binary mask for every image.

## 3. A: Asymmetry

The asymmetry score is computed from the lesion mask only.

The code does the following:

1. Finds the bounding box of the non-zero mask pixels.
2. Expands that region into a square patch around the lesion.
3. Rotates the patch by 180 degrees.
4. Computes the IoU between the original patch and the rotated patch.
5. Converts that into asymmetry using:

$$
A = 1 - \text{IoU}(M, \text{rotate}(M))
$$

where $M$ is the lesion mask patch.

A perfectly symmetric lesion gets a score near 0.
More mismatch between the original and rotated lesion produces a score closer to 1.

In the visualization, this appears as the `Asymmetry diff (abs)` panel.
White regions mark pixels that differ between the original lesion mask and the rotated mask.

## 4. B: Border Irregularity

Border irregularity is based on contour shape.

The code:

1. Extracts the external contour of the binary lesion mask.
2. Selects the largest contour.
3. Computes contour area and perimeter.
4. Calculates circularity using:

$$
\text{circularity} = \frac{4\pi A}{P^2}
$$

5. Converts that to a border irregularity score with:

$$
B = 1 - \text{circularity}
$$

where $A$ is contour area and $P$ is contour perimeter.

More compact, circle-like lesions get lower values.
Jagged, elongated, or irregular borders get higher values.

In the visualization, the `Border + equiv circle` panel shows:

- the lesion contour in green
- an equivalent-area circle in blue

That makes it easy to compare the lesion shape against a regular geometric reference.

## 5. C: Color Variegation

Color variegation is computed inside the lesion mask only.

The current implementation in `scripts/evaluate_abc.py` works like this:

1. Converts the image from BGR to HSV.
2. Keeps only pixels inside the mask.
3. Runs k-means clustering with `k=3` on those HSV pixels.
4. Counts how many clusters are significant, using a 5% proportion threshold.
5. Produces a score based on how many significant clusters are present:

$$
C = \frac{\#\{\text{clusters with proportion} > 0.05\}}{k}
$$

This makes lesions with several distinct color regions score higher than lesions
that are mostly uniform.

In the visualization, the `Color segmentation (kmeans HSV)` panel shows the lesion
recolored by cluster, and the `Cluster palette & proportions` panel shows the cluster
colors and their relative sizes.

## 6. Final ABC Score

The three criteria are combined using a fixed weighted sum:

$$
\text{score} = 0.4A + 0.3B + 0.3C
$$

The asymmetry term has the largest weight, while border and color each contribute
slightly less.

This is the score written to `results_abc.csv`.

## 7. Suspicion Decision

The final suspicion label is binary:

- suspicious if `score >= 0.5`
- not suspicious otherwise

So the score is not just descriptive; it is used as a decision rule.
The threshold can be adjusted depending on whether the goal is to favor recall,
precision, or a balance between both.

The threshold sweep in `threshold_sweep_results.csv` shows how the operating point
changes as the cutoff moves.

## 8. Visualization Workflow

The example panels produced by `scripts/visualize_abc.py` are meant to make the
heuristic interpretable.

Each panel contains:

- Original image
- Mask overlay
- Asymmetry diff
- Border + equivalent circle
- Color segmentation
- Cluster palette and proportions

The set of examples in `visualizations_top20/` can be used directly in slides or
reports to show how ABC behaves on real cases.

## 9. Practical Interpretation

The method is easy to explain because each score is tied to a visual property:

- A rises when the lesion is less symmetric.
- B rises when the border is less regular.
- C rises when the lesion contains more distinct color regions.

The combined score is therefore a heuristic measure of suspicion, not a learned
probability.
